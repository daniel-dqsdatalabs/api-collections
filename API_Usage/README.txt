# Who this collection is for?
This collection is meant to provide administrators, or anyone interested in Postman usage patterns. 

# What does this collection do?
This collection calculates if the Postman feature is over 90% of utilization using tests. This warning is important because features may not be available or teams may incur additional costs if the usage limit is exceeded. 

Insights into feature utiliziation is also available in the visualizations tab. The percentage utiliziation is disaplayed with a radial visualization as seen below.
<img src="https://i.imgur.com/wkliawQ.png"/>

The Postman API utilizations being monitored include:

- [Mock Servers](https://learning.postman.com/docs/designing-and-developing-your-api/mocking-data/mocking-with-examples/)
- [Monitors](https://www.postman.com/api-monitor/)
- [Custom Domains](https://learning.postman.com/docs/publishing-your-api/custom-doc-domains/)
- [Documentation views](https://learning.postman.com/docs/publishing-your-api/viewing-documentation/)
- [Integrations](https://www.postman.com/integrations/)
- API Usage
- Custom Authentication

A notification integration (i.e. [Slack](https://learning.postman.com/docs/integrations/available-integrations/slack/) or [PagerDuty](https://learning.postman.com/docs/integrations/available-integrations/pagerduty/)) can also be set up to inform teams when usage exceeds 90%.
