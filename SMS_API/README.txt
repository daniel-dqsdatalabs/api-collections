# Introduction
What does your API do?
An SMS API. It does the following:
- Create contact
- Delete a contact
- Send message
- Get list of sent messages for a particular contact
- Get list of received messages for a particular conatc
- Delete a particular message

# Overview
Things that the developers should know about

# Authentication
What is the preferred way of using the API?
Not available. It is open to all.

# Error Codes
What errors and status codes can a user expect?
- 409 for conflicts
- 404 when resource cannot be found
- 501 for server side error

# Rate limit
Is there a limit to the number of requests an user can send?
No limit
